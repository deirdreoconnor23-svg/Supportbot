# Password Reset Procedures

## Self-Service Password Reset

### For Users with SSPR Enabled
1. Go to https://passwordreset.microsoftonline.com
2. Enter your username (email address)
3. Complete CAPTCHA
4. Choose verification method:
   - Text to mobile phone
   - Call mobile phone
   - Email to alternate address
   - Security questions
5. Enter verification code
6. Create new password

### Password Requirements
- Minimum 12 characters
- At least one uppercase letter
- At least one lowercase letter
- At least one number
- At least one special character (!@#$%^&*)
- Cannot contain your name or username
- Cannot be any of your last 10 passwords

---

## Account Locked Out

### Cause
Account locks after 5 failed login attempts within 15 minutes.

### Self-Resolution
- Wait 30 minutes - account auto-unlocks
- Or use self-service password reset (resets lockout)

### If User Cannot Self-Resolve
1. Verify user identity (employee ID + DOB or manager confirmation)
2. Open Active Directory Users and Computers
3. Find user account
4. Right-click > Properties > Account tab
5. Click "Unlock account"
6. Click OK

---

## Local Windows Password Reset

### For Users Not on Domain

**If Another Admin Account Exists**
1. Login with admin account
2. Control Panel > User Accounts > Manage another account
3. Select the locked account
4. Click "Change the password"

**Using Password Reset Disk**
1. At login screen, click "Reset password"
2. Insert password reset USB
3. Follow the wizard

**Last Resort - Safe Mode**
1. Restart computer
2. Press F8 repeatedly during boot
3. Select "Safe Mode"
4. Login as hidden Administrator account
5. Reset user password via Control Panel

---

## Multi-Factor Authentication Reset

### User Lost Phone (Authenticator App)
1. Verify identity through alternative method
2. Open Azure AD Admin Center
3. Users > Select user > Authentication methods
4. Delete existing methods
5. User re-registers at https://aka.ms/mfasetup

### User Got New Phone
1. User should transfer authenticator:
   - Old phone: Open Authenticator > Settings > Backup
   - New phone: Install Authenticator > Restore from backup
2. If backup unavailable, follow "Lost Phone" procedure

---

## Service Account Password Changes

### Pre-Change Checklist
- Identify all systems using the account
- Document current password in password vault
- Schedule maintenance window
- Notify affected teams

### Procedure
1. Update password in Active Directory
2. Immediately update password on:
   - Scheduled tasks
   - Windows services
   - IIS application pools
   - SQL Server connections
   - Third-party applications
3. Test all dependent services
4. Update password vault documentation

### Common Gotchas
- Services may cache credentials - restart required
- Some apps need config file updates, not just Windows credential
- Check for hardcoded passwords in scripts

---

## Emergency Admin Password Reset

### Domain Controller
1. Boot into Directory Services Restore Mode (DSRM)
2. Use DSRM password to login
3. Reset admin password via command line:
   ```
   net user Administrator NewPassword123!
   ```
4. Restart normally

### Lost DSRM Password
Contact Microsoft Support - requires proof of ownership and is time-consuming. Always document DSRM passwords in secure vault.
